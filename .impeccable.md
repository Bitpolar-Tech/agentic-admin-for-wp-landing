# Design Context

## Target Audience
WordPress administrators and developers who want an AI-powered assistant for site management. Technical enough to use wp-admin but want natural language interaction. Privacy-conscious — they chose WordPress for control and want AI that respects that philosophy.

## Use Cases
- Discovering the plugin and understanding what it does
- Evaluating privacy/cost benefits vs cloud AI alternatives
- Getting excited enough to try the WordPress Playground demo
- Understanding the technical architecture (local LLM, ReAct reasoning)

## Brand Personality & Tone
- **Confident and direct** — no hedging, no "might help you"
- **Technical credibility** — shows real product screenshots, code examples, accuracy stats
- **Privacy-first gravitas** — serious about data sovereignty, not playful about it
- **Open source pride** — community-driven, transparent, extensible

## Aesthetic Direction
**Editorial tech** — inspired by premium developer tool landing pages (Linear, Vercel, Raycast). Clean editorial layouts with strong typography hierarchy. Dark theme that feels intentional and sophisticated, not "generic dark mode." Asymmetric compositions. Real product screenshots as hero visuals. Restrained color palette with strategic accent moments.

## Differentiation
The "100% local AI" angle is the killer feature. The landing page should make visitors *feel* the privacy promise — not just read about it. The real product screenshots ground the page in reality vs vaporware.
